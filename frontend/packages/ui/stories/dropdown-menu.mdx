# DropdownMenu

Floating menu triggered by a button. Supports action menus, single-select lists, destructive items, and disabled states. Built on Base UI Menu primitives (`@base-ui/react/menu`).

## API

### DropdownMenu (Root)

Wrapper that manages open state and positioning context.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `modal` | `boolean` | `true` | Inherited from Base UI `Menu.Root`. When `true`, locks focus and disables outside interaction while open. Set `false` for non-modal behavior. |

### DropdownMenuTrigger

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `render` | `React.ReactElement` | — | Element to render as the trigger (e.g., wrap a `<Button />`) instead of the default button |

### DropdownMenuContent

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `align` | `"start" \| "center" \| "end"` | `"start"` | Horizontal alignment relative to trigger |
| `side` | `"top" \| "bottom" \| "left" \| "right"` | `"bottom"` | Side to position the menu |
| `sideOffset` | `number` | `4` | Distance from the trigger |

### DropdownMenuItem

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "destructive"` | `"default"` | Visual style (destructive for delete actions) |
| `disabled` | `boolean` | `false` | Disable interaction |
| `inset` | `boolean` | `false` | Add left padding to align with items that have leading icons |

## Upstream Audit

### Primitive library

Restyle of the shadcn **base-nova** dropdown-menu (`components.json` → `"style": "base-nova"`), which ships on Base UI `@base-ui/react/menu` (`Menu as MenuPrimitive`). Base UI is the upstream baseline here — not a SICO divergence. The icon library resolves to `lucide` per `components.json`, so upstream's `IconPlaceholder` lands as `<ChevronRightIcon />` / `<CheckIcon />` on `shadcn add`.

### Component mapping

All 15 upstream exports are preserved 1:1 with the same shadcn names — `DropdownMenu`, `DropdownMenuTrigger`, `DropdownMenuPortal`, `DropdownMenuContent`, `DropdownMenuGroup`, `DropdownMenuLabel`, `DropdownMenuItem`, `DropdownMenuCheckboxItem`, `DropdownMenuRadioGroup`, `DropdownMenuRadioItem`, `DropdownMenuSeparator`, `DropdownMenuShortcut`, `DropdownMenuSub`, `DropdownMenuSubTrigger`, `DropdownMenuSubContent`. No props added, none removed — the public API matches base-nova exactly.

### Documented divergences from upstream

Token swaps (`bg-popover` → `bg-surface-basic`, `text-popover-foreground` → `text-foreground-primary`, `shadow-md`/`shadow-lg` → `shadow-l`, `text-muted-foreground` → `text-foreground-tertiary`, `data-destructive/10` → `button-destructive-*`, focus `accent-foreground` → `foreground-primary`/`foreground-emphasis`) are the restyle itself and live in the **Token Audit** above. The rows below are the non-token-swap divergences.

| # | Divergence | Axis | Rationale |
|---|------------|------|-----------|
| 1 | Custom `dropdown-in/out` keyframes (200ms / 150ms, `cubic-bezier(0.32,0.72,0,1)`) replace base-nova's `data-open:animate-in`/`fade-in-0`/`zoom-in-95`/`data-closed:animate-out`/`fade-out-0`/`zoom-out-95`/`data-[side=*]:slide-in-from-*` + `duration-100` | **className** | SICO motion system |
| 2 | `max-h-97` replaces base-nova `max-h-(--available-height)` | **className** | Cap the panel at 9.5 rows (`h-10` × 9 + 20px + `p-1` × 2 = 388px = `max-h-97`) so the last visible row is clipped in half, cueing scrollability |
| 3 | `border border-divider` replaces base-nova `ring-1 ring-foreground/10` on `DropdownMenuContent` + `DropdownMenuSubContent` | **className** | SICO frames panels with a solid divider border, not a translucent ring |
| 4 | `DropdownMenuSeparator` uses a `border-t` line (`border-divider`) instead of base-nova's `bg-border` fill | **className** | SICO divider technique (matches the panel border) |
| 5 | Enlarged 40px item metrics — `h-10 gap-2 px-3 py-2 rounded-lg leading-body` on `Item`/`SubTrigger`/`CheckboxItem`/`RadioItem` (+`whitespace-nowrap` on `Item`, `px-3 py-2` on `Label`) — vs base-nova `gap-1.5 px-1.5 py-1 rounded-md` | **className** | SICO menu sizing spec |
| 6 | Explicit `text-foreground-secondary` rest color on items (base-nova inherits `popover-foreground` with no explicit rest token) | **className** | Anchors the rest → focus → pressed text progression |
| 7 | `DropdownMenuItem` adds a pressed state — `active:bg-surface-sunken active:text-foreground-emphasis` — that base-nova lacks entirely (its `Item` is focus-fill only) | **className** | Distinct pressed step: fill deepens (`bg-accent` → `bg-surface-sunken`) and text reaches the third color step (`foreground-secondary` → `foreground-primary` → `foreground-emphasis`) |
| 8 | `min-w-24` on `DropdownMenuSubContent` replaces base-nova `min-w-[96px]` (identical 96px, tokenized to the spacing scale) | **className** | Use the scale token instead of an arbitrary value |
| 9 | `cn-rtl-flip` dropped from the `DropdownMenuSubTrigger` chevron | **className** | SICO ships no RTL-flip utility, so the chevron renders unflipped — revisit if RTL support is added |
| 10 | Global `:focus-visible` rule in `globals.css` excludes `[role="menuitem"]`, `[role="menuitemcheckbox"]`, `[role="menuitemradio"]` | **CSS (global)** | Menu items express focus via fill swap (`bg-accent`); the global 2px outline would double-encode focus and flash on every pointer-enter (Base UI moves DOM focus on hover) |
| 11 | `DropdownMenuSubContent` default offsets `alignOffset=-4 sideOffset=4` replace base-nova `alignOffset=-3 sideOffset=0` | **prop default** | SICO nests the submenu with a 4px gap off the parent panel; base-nova opens it flush (`sideOffset=0`) |

### Deleted from upstream

| Removed class | Where | Reason |
|---------------|-------|--------|
| `not-data-[variant=destructive]:focus:**:text-accent-foreground` / `focus:**:text-accent-foreground` | `DropdownMenuItem`, `DropdownMenuSubTrigger`, `CheckboxItem`, `RadioItem` | base-nova forces all descendant text/icons to `accent-foreground` on focus. SICO lets children inherit the item's own focus color so leading icons track the `foreground-secondary` → `accent-foreground` progression. |
| `data-[variant=destructive]:*:[svg]:text-destructive` | `DropdownMenuItem` | base-nova hard-tints destructive icons. SICO lets the icon inherit the item's destructive text color. |
| `dark:data-[variant=destructive]:focus:bg-destructive/20` | `DropdownMenuItem` | Dark-mode destructive fill; SICO has no dark theme yet, so the dark variant is dropped. |
| `cn-menu-target cn-menu-translucent` | `DropdownMenuContent`, `DropdownMenuSubContent` | base-nova menu-translucency infra classes that the SICO design system does not adopt. |
| `duration-100` + `data-[side=*]:slide-in-from-*` + `animate-in`/`animate-out` fade/zoom | `DropdownMenuContent`, `DropdownMenuSubContent` | Replaced by the SICO `dropdown-in/out` keyframes (divergence #1). |

All upstream exports are preserved — these are className-only removals.

### Added to SICO

None. Every export, prop, and structural slot matches base-nova 1:1 — `variant="destructive"`, the trailing (`right-2`) selection indicator, the `RadioItem` checkmark glyph, `inset` on every selectable item, scrollable content, and the `group/dropdown-menu-item` ↔ `DropdownMenuShortcut` focus coupling all ship in base-nova already. The restyle is limited to the token swaps, the className divergences, and the deleted classes above.
