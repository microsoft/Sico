# Textarea

Bare native `<textarea>` styled with SICO tokens. Shadcn restyle of
`base-nova` — only styles diverge from upstream, no upstream drift.

`field-sizing-content` lets the textarea auto-grow with its value — no JS
measurement required. Falls back to `min-h-16` on browsers that don't support
it (Safari < 17.4 / Firefox < 124).

For a char-counter footer, surrounding label, or affix slots, compose with
`<InputGroup>` (via `<InputGroupTextarea>`) and `<Field>` rather than
extending this file.

## Examples

### Sizing & spacing

| Property        | Class                  | Value                                  |
| --------------- | ---------------------- | -------------------------------------- |
| Min height      | `min-h-16`             | 64px                                   |
| Padding (X)     | `px-3`                 | 12px                                   |
| Padding (Y)     | `py-2`                 | 8px                                    |
| Radius          | `rounded-lg`           | 8px                                    |
| Border          | `border`               | 1px                                    |
| Focus shadow    | `shadow-s`             | `var(--shadow-s)`                      |
| Font size       | `text-sm`              | 14px                                   |
| Auto-grow       | `field-sizing-content` | Native CSS — no JS required            |

## Upstream Audit

Shadcn restyle of `base-nova/textarea` (no upstream drift):

- **Styles only** — `border-input` → `border-input-stroke-rest`,
  `bg-background` → `bg-transparent`. Focus uses
  `focus-visible:border-input-stroke-pressed` + `focus-visible:shadow-s`
  (keyboard-only, no ring). Invalid uses
  `aria-invalid:border-input-stroke-error` + `aria-invalid:bg-input-fill-error`.
  Logic and structure untouched.
- **Deleted** — `dark:` variants (SICO has no dark theme yet),
  `md:text-sm` responsive override (SICO uses 14px everywhere).
- **No prop API changes** — accepts all native `<textarea>` props.
