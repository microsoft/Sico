# InputGroup

Form-control shell with addon slots. Drop an `<InputGroupInput>` or
`<InputGroupTextarea>` inside, then add `<InputGroupAddon>` slots on any side
for icons, prefix/suffix text, action buttons, or block-aligned footers. 1:1
port from shadcn `base-nova` — only styles diverge.

Composition contract (no Context, shadcn convention):

- `<InputGroup>` is the shell: border, bg, focus-visible border + shadow,
  data-invalid bg + border. It does not own input state.
- Children declare role via `data-slot`:
  - `[data-slot=input-group-control]` drives focus-visible styling on the shell.
  - `[data-slot=…][aria-invalid=true]` drives the invalid bg + border on the shell.
- `<InputGroupAddon align="…">` positions affixes via `data-align`. The shell
  uses `:has()` selectors to adjust its own layout and the control's padding
  accordingly.

## Token Audit

InputGroup shares the form-control tokens with `<Input>` / `<Textarea>` — the
shell owns the border, focus, and invalid fill + border; the addon owns muted
foreground.

### Shell sizing

| Property      | Class            | Value                                      |
| ------------- | ---------------- | ------------------------------------------ |
| Height        | `h-8`            | 32px (auto when textarea/block addon present) |
| Radius        | `rounded-lg`     | 8px                                        |
| Border        | `border`         | 1px                                        |
| Focus shadow  | `shadow-s`       | `var(--shadow-s)`                          |

### Addon layout

| Align           | Side / placement                | Padding             |
| --------------- | ------------------------------- | ------------------- |
| `inline-start`  | Left of control (default)       | `pl-2`              |
| `inline-end`    | Right of control                | `pr-2`              |
| `block-start`   | Above control (textarea header) | `pt-2 px-2.5`       |
| `block-end`     | Below control (textarea footer) | `pb-2 px-2.5`       |

Inline addons pull the control's padding via `:has()` so icons sit flush:
`pl-1.5` when leading, `pr-1.5` when trailing.

### Button size scale

`<InputGroupButton>` is a wrapped `<Button>` with custom sizes tuned for the
24px addon row.

| Size       | Class          | Dimensions    | Use                  |
| ---------- | -------------- | ------------- | -------------------- |
| `xs`       | `h-6 px-1.5`   | 24px tall     | Text actions         |
| `sm`       | inherited      | Button default| Larger text actions  |
| `icon-xs`  | `size-6`       | 24 × 24       | Square icon actions  |
| `icon-sm`  | `size-8`       | 32 × 32       | Larger icon actions  |

## Upstream Audit

Shadcn restyle of `base-nova/input-group` (no upstream drift):

### Token swaps

- `border-input` → `border-input-stroke-rest`.
- Focus styling uses `focus-visible:` on the shell's `:has()` selector for
  the control — `border-input-stroke-pressed` + `shadow-s`, no ring.
- Invalid styling uses `border-input-stroke-error` + `bg-input-fill-error`,
  also no ring. The inner controls (`InputGroupInput` / `InputGroupTextarea`)
  strip their own `aria-invalid:` border/bg/ring so error visuals come from
  the shell only.
- `text-muted-foreground` → `text-foreground-tertiary`.

### API contract

- No prop API changes — same `align` / `size` / composition contract.
- `InputGroupButton` defaults to `variant="subtle"` as the closest SICO
  equivalent to shadcn's `ghost`.
- Layout cva variants are kept verbatim.

### Deleted from upstream

| Deleted | Reason |
| ------- | ------ |
| `[&>kbd]:rounded-[calc(var(--radius)-5px)]` | SICO has no `--radius` token and no `<kbd>` UI yet; use `rounded-md` |
| `in-data-[slot=combobox-content]:*` | Combobox not yet in SICO |
| `dark:` variants | SICO has no dark theme |
| `InputGroupAddon` click-to-focus | SICO leaves click-to-focus to consumers |
