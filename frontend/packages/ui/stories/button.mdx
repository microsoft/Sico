# Button

An interactive control that triggers an action. Built on
[`@base-ui-components/react`](https://base-ui.com/react) with SICO design tokens.
Shadcn restyle of `base-nova` — every divergence from upstream (token swaps,
naming, spacing, radius, disabled treatment, added `aria-invalid` styling) is
enumerated in § Upstream Audit.

### Icons

Tag the icon element with `data-icon="inline-start"` or `data-icon="inline-end"`
(matching the position relative to the label). Icons get a default `size-4`
(16px) via `[&_svg:not([class*='size-'])]:size-4` in the variant base class —
uniform across **every** size (the container scales, the glyph does not). Never
size icons manually inside Button; an icon carrying its own `size-*` opts out of
the base rule (the `:not([class*='size-'])` guard) and keeps its own size.

### Sizing & spacing

Padding compresses by ~4px on the side that carries an icon —
`has-data-[icon=inline-start]:p[lr]-*` / `has-data-[icon=inline-end]:p[lr]-*`.
`icon-*` sizes are square and ignore padding.

| Size       | Class    | Size  | Gap     | Padding  | Padding (with icon) | Glyph  |
| ---------- | -------- | ----- | ------- | -------- | ------------------- | ------ |
| `xs`       | `h-6`    | 24px  | `gap-1`   | `px-2`    | `px-1.5` (icon side) | 16px |
| `sm`       | `h-7`    | 28px  | `gap-1`   | `px-2.5`  | `px-1.5` (icon side) | 16px |
| `default`  | `h-8`    | 32px  | `gap-1.5` | `px-3`    | `px-2` (icon side)   | 16px |
| `lg`       | `h-9`    | 36px  | `gap-1.5` | `px-3`    | `px-2` (icon side)   | 16px |
| `icon-xs`  | `size-6` | 24px  | —         | —         | —                    | 16px |
| `icon-sm`  | `size-7` | 28px  | —         | —         | —                    | 16px |
| `icon`     | `size-8` | 32px  | —         | —         | —                    | 16px |
| `icon-lg`  | `size-9` | 36px  | —         | —         | —                    | 16px |

**Special cases:**

- `link` variant hardcodes `h-6` (24px) and ignores the `size` prop — link
  buttons always render at 24px regardless of the size value passed.
- Inside a `<div data-slot="button-group">`, the `xs`/`sm`/`icon-xs`/`icon-sm`
  sizes switch from `rounded-md` (6px) to `rounded-lg` (8px) via
  `in-data-[slot=button-group]:rounded-lg`.

### Typography

All sizes inherit `text-sm` from the cva base class (14px). `xs` overrides to
`text-xs` (12px) for dense toolbars.

| Size                  | Class     | Value | Source            |
| --------------------- | --------- | ----- | ----------------- |
| `xs`                  | `text-xs` | 12px  | size variant      |
| `sm`/`default`/`lg`   | `text-sm` | 14px  | cva base class    |

### Border radius

| Sizes                                      | Class        | Value |
| ------------------------------------------ | ------------ | ----- |
| `xs`, `sm`, `icon-xs`, `icon-sm`           | `rounded-md` | 6px   |
| `default`, `lg`, `icon`, `icon-lg`         | `rounded-lg` | 8px   |

### Shadow tokens

Applied as `shadow-button-*` classes on the variants that need surface
elevation. All three resolve to `--shadow-s` today and are removed by
`disabled:shadow-none`.

| Token                                | Used by                | Resolves to    |
| ------------------------------------ | ---------------------- | -------------- |
| `--shadow-button-primary`            | `primary`              | `--shadow-s`   |
| `--shadow-button-secondary`          | `secondary`            | `--shadow-s`   |
| `--shadow-button-destructive-outline`| `destructive-outline`  | `--shadow-s`   |

## Upstream Audit

This component is a shadcn restyle of `base-nova/button`. The component logic,
ARIA behavior, and JSX shape match upstream; styling, naming, and a small set of
appended utilities diverge — every divergence is enumerated below.

### Token swaps

Every upstream className that resolves to a shadcn default is swapped for the
matching SICO token. The full color / fill / stroke mapping lives in the Token
Audit above; this table covers the non-color swaps and the cva key renames.

| Upstream                                                            | SICO replacement                                                                                              |
| ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `bg-primary` / `bg-secondary` / `bg-destructive` / etc.             | `bg-button-{variant}-fill-rest` (see Token Audit for full color mapping)                                      |
| variant keys: `default` / `ghost`                                   | `primary` / `subtle` (semantic naming — `primary` reads as action hierarchy; `subtle` describes the treatment) |
| `default`/`lg` padding `px-2.5`                                     | `px-3` (SICO spacing scale anchors text buttons at 12px horizontal padding)                                   |
| `xs`/`sm`/`icon-xs`/`icon-sm` `rounded-[min(var(--radius-md),Npx)]` | `rounded-md` (6px — SICO radius scale uses fixed steps, not upstream's pixel-clamp)                           |
| `sm` text `text-[0.8rem]` (12.8px)                                  | inherits `text-sm` (14px) from base — SICO type scale has no 12.8px step                                      |
| `disabled:opacity-50`                                               | per-variant `disabled:bg-*-fill-disabled` + `disabled:text-*-foreground-disabled` (`link` keeps text-only disabled) |
| `link` focus ring (shared upstream ring)                            | `focus-visible:border-transparent focus-visible:ring-0` — link renders inline with text                       |
| _(SICO addition)_ press affordance                                  | `active:not-aria-[haspopup]:translate-y-px` — pressed buttons nudge down 1px, except when they're popover triggers (`aria-haspopup`)  |

### API contract

The prop surface matches upstream verbatim. `Button` accepts every prop forwarded
to `@base-ui-components/react`'s `ButtonPrimitive` — `variant`, `size`,
`disabled`, `className`, plus all native button attributes and event handlers.
The `variant` / `size` unions accept SICO-renamed values (`primary` / `subtle`
instead of `default` / `ghost`; extra `xs`, `icon-xs`, `icon-sm`, `icon-lg` sizes
and the SICO-only `destructive-outline` variant) but the prop shape, event
semantics, ARIA behavior, focus management, and the `data-icon="inline-start" |
"inline-end"` slot convention all match upstream. The rendered tree adds only
`data-slot="button"`.

### Deleted from upstream

| Deleted                            | Reason                                                                                          |
| ---------------------------------- | ----------------------------------------------------------------------------------------------- |
| `dark:aria-invalid:*` utilities    | SICO has no dark-mode theme yet — same omission across `Input`, `Textarea`, `InputGroup`        |
