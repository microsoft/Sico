# Dialog

Modal dialog for confirmations, forms, and content. Built on Base UI Dialog
primitive with SICO design tokens. Shadcn restyle — upstream reference: [shadcn/ui dialog](https://ui.shadcn.com/docs/components/dialog).

## Examples

### Confirmation

`variant="confirmation"` size preset.

## Upstream Audit

Restyle of `base-nova/dialog`, built on `@base-ui/react/dialog`. Divergence from
upstream is className only — token swaps plus a dropped footer bar; the footer's
flex layout, primitive library, props, structure, and accessibility match
upstream.

### Token swaps

Per `npx shadcn@latest diff dialog` against `base-nova/dialog`:

| Upstream class | SICO token |
|----------------|------------|
| `bg-popover` | `bg-surface-basic` |
| `text-popover-foreground` | `text-foreground-primary` |
| `ring-1 ring-foreground/10` | `border border-divider` |
| `bg-black/10` (backdrop) | `bg-overlay-black-50` |
| `text-muted-foreground` (description) | `text-foreground-secondary` |
| description link `*:[a]:hover:text-foreground` | `*:[a]:hover:text-foreground-primary` (the `*:[a]:underline *:[a]:underline-offset-3` affordance is kept verbatim) |
| close button `variant="ghost"` at `top-2 right-2` | `variant="subtle"` at `top-5 right-5` (aligns to the `p-5` padding) |
| `DialogFooter` close button `variant="outline"` | `variant="secondary"` (SICO has no `outline` Button; `secondary` is the neutral-bordered equivalent) |

### API contract

| Prop | Upstream | SICO | Notes |
|------|----------|------|-------|
| `open` | ✓ | ✓ | Same |
| `onOpenChange` | ✓ | ✓ | Same |
| `DialogContent` `showCloseButton` | ✓ | ✓ | Upstream prop, preserved (default `true`) |
| `DialogFooter` `showCloseButton` | ✓ | ✓ | Upstream prop, preserved (default `false`); appends a built-in `Close` button, restyled `outline`→`secondary` |
| `DialogContent` `variant` | — | ✓ | Added: `"confirmation"` (default) or `"content"` size preset, driven by the `dialogContentVariants` cva |

### Layout drift (design-driven, not token swaps)

The footer's flex layout matches upstream verbatim
(`flex flex-col-reverse gap-2 sm:flex-row sm:justify-end`) — including the
mobile-stack-to-row reflow. The one divergence is the dropped tinted bar:

| Dropped upstream classes | Reason |
|--------------------------|--------|
| `-mx-4 -mb-4 border-t bg-muted/50 rounded-b-xl p-4` (the tinted footer bar) | Figma footer sits flush inside the dialog — no tinted bar, no divider, no edge-bleed. The bar's `-mx`/`-mb`/`rounded-b`/`p` are scaffolding that only exist to bleed that bar to the dialog edge, so they go with it. |

### Deleted from upstream

None — every upstream prop and exported part is preserved.
