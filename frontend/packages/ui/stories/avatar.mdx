# Avatar

User-image shell built on Base UI `@base-ui/react/avatar`. Drop an
`<AvatarImage>` for the picture and an `<AvatarFallback>` for the
initials/icon backup â€” Base UI swaps between them automatically based
on image loading status. Stack avatars with `<AvatarGroup>` +
`<AvatarGroupCount>`, overlay a status dot with `<AvatarBadge>`. 1:1
port from shadcn `base-nova/avatar` â€” `size` extended with `xs` /
`2xl` for SICO use cases.

Composition contract (no Context, shadcn convention):

- `<Avatar>` is the shell: rounded square, `data-size`, `after:*`
  outline.
- `<AvatarImage>` renders the picture. Base UI handles load /
  error swap.
- `<AvatarFallback>` renders the initials / icon when the image is
  missing or failed.
- `<AvatarBadge>` is positioned absolute bottom-right; sized via
  `group-data-[size=*]/avatar:*` from the parent.
- `<AvatarGroup>` stacks avatars and applies a 2px ring in
  `--color-surface-basic` to every child for visual separation.

> **Fallback contract.** Every `<Avatar>` **must** contain an
> `<AvatarFallback>` child. Without it the shell collapses to an
> empty rounded box during image loading / error. Two patterns:
>
> - Real users â†’ text initials: `<AvatarFallback>JD</AvatarFallback>`
> - Digital Workers â†’ default SVG:
>   `<AvatarFallback><img src={DW_DEFAULT_AVATAR_URL} alt="" /></AvatarFallback>`
>
> Enforced at lint via `local-rules/require-avatar-fallback`.

## Upstream Audit

Shadcn restyle of `base-nova/avatar` (no upstream drift):

- **Styles only** â€” every shadcn token swapped to SICO semantic
  equivalents (see "Deleted from upstream" table). JSX, cva structure,
  composition, accessibility kept verbatim.
- **`size` extended with `xs` and `2xl`** â€” SICO product surfaces
  (dense mention lists at 20px, hero profile cards at 48px). Upstream
  ships `default` / `sm` / `lg` only. `AvatarBadge` was extended to
  cover the new sizes with a linear-interpolated scale (`xs:size-1.5`,
  `2xl:size-3.5`); `AvatarGroupCount` mirrors the Avatar shell scale
  (`xs:size-5`, `2xl:size-12`) so stacked groups stay symmetric.
- **No prop API changes** â€” every other prop matches upstream verbatim.
- **No JSX changes** â€” `<AvatarFallback>` transparently passes
  `children` through; no default icon injected.
- **No behavior changes** â€” image loading swap is owned by Base UI.
